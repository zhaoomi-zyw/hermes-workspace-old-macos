# Agent Workshop for a German High School Intern

Use this reference when designing a beginner-friendly AI Agent / Hermes workshop for an international high school student, especially a German student visiting China for a short internship.

## Audience

- German high school student.
- Temporarily in China for a short internship.
- Exact interests may be unknown before the session.
- Likely needs concrete, cross-cultural examples rather than abstract architecture.
- Working discussion with Omi may happen in Chinese, but student-facing output should be English.

## Confirmed workshop shape

- Duration: 2 hours.
- Style: balanced.
  - First: spark curiosity and explain why agents matter.
  - Middle: hands-on experience with a visible artifact.
  - End: connect agent literacy to future study, university choices, and careers.
- Final courseware format: multiple HTML files, not one monolithic slide deck.
- Student takeaways:
  1. One small HTML artifact created during the workshop.
  2. A reusable set of AI Agent prompt templates.

## Core message

Do not start with jargon. Use this contrast:

- A chatbot answers questions.
- An AI agent completes tasks.

Helpful analogy:

- ChatGPT is like a smart person inside a chat window.
- Hermes-like agents are like smart assistants sitting at a computer: they can use tools, search, write files, run code, remember preferences, and automate recurring work.

## Recommended 2-hour agenda

1. Interest discovery, 10 min
   - Ask what he would want a computer-using AI assistant to do.
   - Offer categories: school, AI/coding, China internship, football/sports, gaming/esports, future career.
   - Use his answer to choose the live demo path.

2. Chatbot vs Agent concept, 20 min
   - Explain tools, files, browser/search, coding, memory, and automation in everyday language.
   - Avoid gateway, MCP, provider setup, profiles, and cron internals at this stage.

3. Live demo 1: idea to artifact, 25 min
   - Default artifact options:
     - My China Internship Diary
     - personal mini website
     - simple browser game
     - football/esports interest page
     - AI learning portfolio
   - Show the agent creating and revising a real file.

4. Live demo 2: study or research assistant, 20 min
   - Possible topics:
     - How should a German high school student start learning AI?
     - What careers will AI agents change?
     - How can I summarize my China internship experience?
   - Show structuring, summarizing, and adapting explanation level.

5. Student hands-on challenge, 25 min
   - Let the student choose one challenge.
   - Omi uses Hermes to guide him through the first version.

6. Future trend discussion, 10 min
   - Search literacy mattered in the 2000s.
   - Prompt literacy matters now.
   - Agent literacy will matter next: learning to delegate tasks to AI systems.

7. Next steps, 10 min
   - First experience: Omi's already configured Hermes WebUI.
   - If interested later: VS Code + Hermes CLI.
   - Defer advanced setup until after motivation is established.

## Interest-to-demo mapping

- Coding / AI / technology → mini website or browser game.
- School / exams → AI study coach.
- China internship → China internship diary or survival assistant.
- Football / sports → match or player analysis assistant.
- Gaming / esports → gaming stats or strategy assistant.
- Future career → AI career exploration plan.

## Suggested multi-file HTML package

Use multiple student-facing English HTML files:

1. `index.html` — main 2-hour workshop guide.
2. `chatbot-vs-agent.html` — visual comparison page.
3. `live-demo-guide.html` — adaptive demo routes based on interests.
4. `agent-prompt-templates.html` — reusable prompts for the student.
5. `student-next-steps.html` — follow-up learning and installation path.

## Prompt template handout categories

- Study coach.
- Research assistant.
- Career explorer.
- China internship diary helper.
- Coding buddy.

## Visual design notes

For final HTML courseware, combine:

- `educational-workshop-courseware` for teaching structure.
- `claude-design` for artifact design process and taste.
- `design-html` for final HTML/CSS usability and visual polish when available.

Prefer a modern, professional tech style that does not feel childish. Keep the pages readable for a high school student and suitable for an internship environment.
